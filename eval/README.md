# Evaluation and Post-Processing

This directory contains the post-processing and evaluation pipeline for
predictions generated by [`../test_slam.py`](../test_slam.py).

The workflow converts predicted and ground-truth polar representations into
Cartesian images and point clouds, then performs qualitative and quantitative
comparison.

## Expected input

Running:

```bash
python3 test_slam.py
```

from the repository root should create predicted radar outputs and corresponding
LiDAR ground truth in a run-specific test-image directory, typically under
`logs/<run_id>/`.

The exact directory name depends on the configuration used by `test_slam.py`.

## Complete pipeline

From the repository root:

```bash
cd eval
python3 postprocess_slam.py
```

`postprocess_slam.py` is intended to orchestrate the individual conversion and
evaluation stages. Verify its configured input run directory and output paths
before execution.

## Step-by-step pipeline

### 1. Polar-to-Cartesian conversion

```bash
python3 pol_to_cart_slam.py
```

This stage converts polar model outputs and ground-truth images into Cartesian
representations.

A typical output directory may follow a pattern such as:

```text
processed_imgs_<run_id>_test_imgs/
```

### 2. Cartesian image to point cloud

```bash
python3 image_to_pcd_slam.py
```

This stage converts the Cartesian representations into point-cloud files used
for visualization and geometric error evaluation.

### 3. Quantitative comparison

```bash
python3 pc_compare.py
```

This stage computes the configured point-cloud comparison metrics and may
produce cumulative-distribution plots or summary files.

The project documentation refers to Chamfer- and Hausdorff-based comparisons.
Confirm the exact metric implementation and spelling in the source before
reporting results.

### 4. Qualitative visualization

From the repository root:

```bash
python3 pcd_visualize.py
```

This utility visualizes generated and reference point clouds for qualitative
comparison.

## Output handling

Evaluation outputs may include:

- Cartesian images;
- predicted point clouds;
- reference point clouds;
- per-sample distances;
- cumulative-distribution plots;
- summary metrics.

These outputs should normally remain outside ordinary Git tracking. Before
publishing any figure or point cloud, verify that it does not disclose
restricted SLAM-RF data or an unauthorized derivative.

## Reproducibility notes

The numerical results depend on:

- the selected checkpoint;
- preprocessing and clipping parameters;
- coordinate conventions;
- polar-to-Cartesian interpolation;
- point-cloud thresholding;
- spatial alignment;
- metric implementation and units.

Record all relevant settings together with each evaluation result.
